# Client Delivery System
*Version 1.0 — Initial draft. This document is a living system and will be updated as the platform evolves.*

---

## Overview
The LESARUSS delivery model is built around a progressive door system. Every client enters through the same front door — but where they go next depends on what they bring, what they need, and what tier they are on. Each door collects exactly what is needed for that phase. Nothing more. Nothing less. When a door closes behind them, the system has everything it needs to move to the next phase.

This is not a linear checklist. It is a pipeline. Each phase feeds the next, and every completed client journey feeds back into the LESARUSS operating system — making the next version stronger for everyone.

---

## The Door Model

### Door 1 — The Front Door (Big Picture Intake)
**Trigger:** Client submits initial intake form at any tier.
**Purpose:** Capture the big picture. Who are they, where are they, and what are they building?

**What this door collects:**
- Brand name, industry, and niche
- Current stage (idea, early, established, scaling)
- What they've tried and what hasn't worked
- Their primary goal in the next 90 days
- Which tier they are entering at (DIY / Guided / Done For You)
- Any existing assets (website, social, content, offers)

**Output:** Client profile is created in the system. Tier is confirmed. They are routed to Door 2.

---

### Door 2 — The Deep Dive (System Audit)
**Trigger:** Door 1 is complete. Tier determines depth of this phase.
**Purpose:** Get specific. This door looks at the current state of every system: brand, messaging, offer, content, delivery, and operations.

**What this door collects (by section):**
- **Brand:** Core message, positioning, audience, differentiator
- **Offer:** What they sell, how they price it, how they deliver it
- **Content:** Current channels, posting cadence, content types, performance
- **Operations:** Tools in use, team structure, workflows
- **Gaps:** Where the client knows they are weak or undefined

**Tier variance:**
- **DIY ($11/mo):** Self-guided audit via structured questionnaire. Client completes on their own.
- **Guided ($111/mo):** Audit completed via guided session with a LESARUSS operator.
- **Done For You ($10k/quarter):** Full audit conducted by LESARUSS team through direct research and client interview.

**Output:** System gap report generated. Blueprint build begins. Client routed to Door 3.

---

### Door 3 — The Blueprint Build
**Trigger:** System audit is complete.
**Purpose:** Build the client's operating system using the LESARUSS framework. This is where the system becomes theirs.

**What gets built:**
- Brand Foundation document (tailored to client)
- Messaging and Content System
- Offer and Monetization Map
- Content Execution Plan
- Delivery Workflow
- Community/Audience System (if applicable)
- 90-Day Operating Plan

**Tier variance:**
- **DIY:** Client receives templates and builds using LESARUSS documentation. Self-service with access to community.
- **Guided:** LESARUSS operator works alongside client to co-build each component.
- **Done For You:** LESARUSS team builds everything. Client reviews, approves, and receives the completed system.

**Output:** Client operating system (V1) is delivered. Onboarding to live system begins.

---

### Door 4 — Activation and Operations
**Trigger:** Blueprint is approved and handed off.
**Purpose:** The system goes live. Client begins operating from their LESARUSS blueprint.

**What happens here:**
- System is activated across relevant platforms and tools
- Operating cadence is established (weekly, monthly rhythms)
- Performance baselines are set
- Client begins generating data and output from inside the system

**Tier variance:**
- **DIY:** Client activates on their own. Responsible for following the system and tracking their own metrics.
- **Guided:** Check-in sessions to ensure activation is clean and the system is running properly.
- **Done For You:** LESARUSS manages full activation, platform setup, and initial operating cycle.

**Output:** Client is live. System data begins feeding back into LESARUSS platform for version improvement.

---

### Door 5 — Upgrade and Evolution
**Trigger:** New LESARUSS system version is released OR client hits a defined growth threshold.
**Purpose:** Keep the system current. Every version upgrade improves the system based on collective client data.

**What happens here:**
- LESARUSS releases a new system version (V2, V3, etc.)
- All users are notified of what changed and why
- Upgrade path is defined for each tier

**Tier variance:**
- **DIY ($11/mo):** Client is responsible for reviewing and applying upgrades to their own system. Documentation provided.
- **Guided ($111/mo):** Guided session to walk through what changed and how to apply it.
- **Done For You ($10k/quarter):** LESARUSS handles all upgrades. Client is notified of what changed, not asked to do the work.

**Output:** Client system is updated to the latest version. Cycle continues.

---

## System Feedback Loop
Every client who completes a phase contributes data to the LESARUSS platform. Patterns across clients — what works, what gaps appear most often, what questions come up at each door — are used to improve the system for the next version. This is how LESARUSS compounds in value over time. The more clients in the system, the smarter the system gets.

---

## Success Metrics
- **Door Completion Rate** — What % of clients move cleanly through each door?
- **Time to Blueprint** — How long from Door 1 intake to completed Blueprint?
- **Activation Rate** — What % of clients fully activate their system?
- **Upgrade Adoption** — What % of users apply new version updates?
- **Client System Performance** — Are clients' systems producing results? (tracked by tier)
